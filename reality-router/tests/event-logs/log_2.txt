    0 2 * * * /bin/bash -c 'cd /path/to/your/PolyMarket && source venv/bin/activate && python main.py \\u003e\\u003e /path/to/your/PolyMarket/cron.log 2\\u003e&1'\

# Project Setup and Daily Execution\
\
This document outlines the steps to set up the project locally and configure it for daily automated execution.\
\
## 1. Local Setup\
\
### 1.1. Python Environment and Dependencies\
\
1.  **Create a Python Virtual Environment**:\
    ```bash\
    python3 -m venv venv\
    ```\
\
2.  **Activate the Virtual Environment**:\
    -   On macOS/Linux:\
        ```bash\
        source venv/bin/activate\
        ```\
    -   On Windows:\
        ```bash\
        .\\\\venv\\\\Scripts\\\\activate\
        ```\
\
3.  **Install Dependencies**:\
    ```bash\
    pip install -r requirements.txt\
    ```\
\
### 1.2. Environment Variables\
\
Create a [`.env`](.env.template) file in the project root based on the provided [`.env.template`](.env.template) file. This file should contain your `DATABASE_URL`.\
\
Example `.env` file content:\
```\
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/polymarket_sports\
```\
\
### 1.3. Starting the Postgres Database\
\
This project uses Docker Compose to manage the PostgreSQL database.\
\
1.  **Ensure Docker is Running**: Make sure Docker Desktop or your Docker daemon is running on your system.\
\
2.  **Start the Database**: Navigate to the project root directory and run:\
    ```bash\
    docker compose up -d\
    ```\
    This command will start the PostgreSQL container in the background.\
\
## 2. Daily Execution Orchestration\
\
The `main.py` script needs to be executed once a day to capture market snapshots and update game outcomes. We recommend using `cron` for scheduling on Linux/macOS systems.\
\
### 2.1. Scheduling with Cron (Linux/macOS)\
\
1.  **Open Crontab Editor**:\
    ```bash\
    crontab -e\
    ```\
    This will open your user's crontab file in a text editor.\
\
2.  **Add a Cron Job Entry**:\
    Add the following line to the crontab file to execute [main.py](main.py) daily at 2:00 AM. Ensure you replace `/path/to/your/PolyMarket` with the actual absolute path to your project directory.\
\
    ```cron\
    0 2 * * * /bin/bash -c 'cd /path/to/your/PolyMarket && source venv/bin/activate && python main.py \\u003e\\u003e /path/to/your/PolyMarket/cron.log 2\\u003e&1'\
    ```\
\
    **Explanation of the Cron Entry**:\
    -   `0 2 * * *`: This is the cron expression, meaning \\\"at 02:00 AM every day.\\\"\
        -   `0`: Minute (0-59)\
        -   `2`: Hour (0-23)\
        -   `*`: Day of month (1-31)\
        -   `*`: Month (1-12)\
        -   `*`: Day of week (0-7, where 0 and 7 are Sunday)\
    -   `/bin/bash -c '...'`: Executes the commands within the single quotes using bash.\
    -   `cd /path/to/your/PolyMarket`: Changes the current directory to your project's root. **Crucial for cron to find your files.**\
    -   `source venv/bin/activate`: Activates the Python virtual environment. This ensures all project dependencies are available.\
    -   `python main.py`: Executes the main script.\
    -   `\\u003e\\u003e /path/to/your/PolyMarket/cron.log 2\\u003e&1`: Redirects both standard output and standard error to a log file named `cron.log` within your project directory. This is essential for debugging.\
\
3.  **Save and Exit**: Save the crontab file and exit the editor. The cron job will be scheduled automatically.\
\
### 2.2. Verifying Cron Jobs\
\
You can list your active cron jobs using:\
```bash\
crontab -l\
```\
\
After the scheduled time, check the [cron.log](cron.log) file in your project directory for output and any potential errors.\

### 1.2. Starting the Application\
\
This project uses Docker Compose to manage both the PostgreSQL database and the Python Python data extractor. The extractor includes its own internal scheduling loop (defined in [`run_extractor.sh`](run_extractor.sh)), running periodically without the need for host-level cron jobs.\
\
1.  **Ensure Docker is Running**: Make sure Docker Desktop or your Docker daemon is running on your system.\
\
2.  **Start Services**: Navigate to the project root directory and run:\
    ```bash\
    docker compose up -d\
    ```\
    This command will build the extractor image (if necessary) and start both the PostgreSQL and extractor containers in the background.\
\
## 2. Execution and Logs\
\
The data extractor runs automatically as part of the containerized setup. It executes `main.py` to capture market snapshots and update game outcomes, then sleeps for a defined period (e.g., 1 hour) before running again, completely replacing the former cron setup.\
\
You can view the logs of the extractor at any time to verify it's working correctly:\
```bash\
docker compose logs -f extractor\
```\
\
To stop the entire application, simply run:\
```bash\
docker compose down\
